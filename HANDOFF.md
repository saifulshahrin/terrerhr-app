# Terrer Handoff Notes

When working in this repo, Codex must update STATUS.md during implementation.

Required updates:
1. mark current task being worked on
2. log each completed implementation step
3. list files changed
4. note any blocker or risk immediately
5. note validation results after each major step

If blocked or uncertain:
- stop coding
- write the blocker clearly in STATUS.md
- do not expand scope

At the end, write:
- what was completed
- what remains
- what should be tested in the app by the owner
